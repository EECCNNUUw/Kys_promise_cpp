## 📖 项目概述

本项目是一个极具挑战性的工程实践：将经典的《金庸群侠传前传》（基于 Pascal/Delphi 开发）通过 **C++20** 进行全方位的现代化重构。这不仅仅是代码语言的迁移，更是一次底层架构的升华与性能的极致压榨。

我们的目标是在保持原版逻辑（Binary Compatibility）百分之百还原的基础上，引入现代化的游戏开发范式，彻底解决原版引擎在高性能渲染、跨平台扩展以及复杂内存管理上的历史局限。

## 🚀 核心特性与技术栈

-   **高性能渲染引擎**：基于 **SDL3** 深度定制，通过 GPU 加速实现丝滑般的 60FPS 画面表现，彻底告别原版的渲染瓶颈。
-   **现代 C++ 架构**：采用单例管理（GameManager）、事件驱动架构（EventManager）以及高度解耦的 UI 系统（UIManager），使代码具备极高的可读性与可维护性。
-   **二进制数据兼容**：自研数据解析层，直接完美读取原版 `.grp`、`.idx`、`.pic` 等二进制资源，实现逻辑上的无缝衔接。
-   **繁体编码深度适配**：内置高性能文本渲染引擎，完美兼容繁体中文（Big5/UTF-8）及特殊字体渲染，重现武侠原味。
-   **现代化 UI 重建**：通过数学建模重构环形菜单（Ring Menu）及各类次级界面，支持动态高亮与平滑交互。

## 🛠️ 重构进展 (Project Roadmap)

目前项目正处于高速开发阶段，已完成核心骨架的搭建：

-   [x] **Core System**: 基础图形库集成、全局状态机管理。
-   [x] **UI Framework**: 环形 ESC 主菜单、武学/状态/系统/技能/队友/物品六大次级菜单逻辑重构。
-   [x] **Data Engine**: 二进制资源读取、贴图切片算法校准。
-   [ ] **Battle Logic**: 正在进行中，计划引入更智能的寻路算法与战斗平衡。
-   [ ] **Event System**: 复杂脚本引擎的 C++ 移植与优化。

## 💎 为什么选择重构？

原版的 Pascal 代码虽然经典，但在现代硬件环境下难以进行深度二次开发。通过 C++ 重构，我们获得了：
1.  **跨平台潜力**：不仅限于 Windows，未来可轻松扩展至 Linux、macOS 乃至移动端。
2.  **极致性能**：利用 C++ 的零成本抽象（Zero-cost Abstractions）和精细内存控制，为后续大规模 MOD 扩展提供空间。
3.  **开发者友好**：现代化的工程化配置（CMake），让每一个有志于武侠游戏开发的极客都能轻松上手。
